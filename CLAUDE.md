# Проект «Pentatonic Trainer» — брифинг v1

## Текущий фокус

**Приоритет:** реализована фича «Гаммы / лады» (пункт 3 плана развития) — новый блок между выбором бокса и грифом: переключатель «Пентатоника / +лад», добавленные ноты гаммы поверх пентатоники (фиолетовые), характерная нота лада (оранжевая) с подсказкой, карточка с нотами + жанровой привязкой. 7 ладов: минор → эолийский/дорийский/фригийский/локрийский, мажор → ионийский/миксолидийский/лидийский. Всё проверено программно (jsdom, ~30 assert'ов + проверка, что лады — корректные ротации мажорной гаммы), НЕ на глаз. Задеплоено: файл в GitHub → Netlify авто-деплой (`main@d66b35c`). Денис подтвердил визуально — фича на месте на живом сайте.

**Что осталось (план развития, по убыванию ценности за усилия):**

1. **Блюзовая нота (b5) как переключатель** — одна кнопка рядом с MINOR/MAJOR, для блюзовой импровизации нота №1, стоит копейки (один интервал + цвет точки).
2. **Ритм-секция: бас + бочка/снейр** — сейчас джем-луп это только бой аккордами. Бас на тонике-квинте по долям + примитивная бочка (синус с быстрым спадом) + снейр (шум) превратят луп в настоящий backing track. Web Audio без семплов, движок уже есть. Самое большое качественное изменение для джема.
3. ✅ **Гаммы / лады** — СДЕЛАНО (см. Текущий фокус). Остаётся визуальная приёмка плотности ALL-режима с ладом на телефоне.
4. **Лики / ходы / «вертушки» как подсветка на грифе** — не библиотека табов, а 6-10 визуальных паттернов, подсвечиваемых по очереди в такт (call & response в боксе 1, блюзовая фраза с бендом b3→3, пентатоническая «шестёрка», ход между боксами через общую ноту, регги/фанк муто-риф). Привязка к прогрессиям, запуск кнопкой ▶ рядом. НЕ превращать в 50 ликов — путь к перегрузке.
5. **Точечное добавление прогрессий** — дорийский фанк (i–II, Oye Como Va), 16-тактовый блюз, соул-баллада i–III–VII–iv. 5-7 штук, не больше.
6. **Точечное добавление ритмов** — 6/8 баллада, простые четверти для новичкового темпа.

**Найденные баги (из ревизии, ещё НЕ исправлены — порядок по слышимости):**

1. 🔴 **Регистровые скачки в прогрессиях** — корень считается `48 + ((ri + d) % 12)`, все корни зажаты в октаву C3–B3. В Am красиво (A–G–F–E вниз), но в Em/Cm нисходящая андалузская каденция ломается: V прыгает вверх на септиму вместо шага вниз. Лечится выбором ближайшей октавы к предыдущему аккорду (примитивный voice leading, ~10 строк). **Самое слышимое — чинить первым.**
2. 🟡 **SVG-рамки боксов не пересчитываются при ресайзе/повороте** — `drawBoxOverlay` вызывается один раз через setTimeout. Повернул телефон — рамки съехали. Нужен слушатель `resize`/`orientationchange`.
3. 🟡 **Громкость до 1.5 ПОСЛЕ лимитера** — `masterOutNode` умножает на 1.5 уже после компрессора, на максимуме полифония с дисторшном может клиповать с треском. Слайдер до компрессора либо максимум 1.0.
4. 🟡 **Karplus-Strong генерируется синхронно на каждый штрих** — 16-й фанковый паттерн = ~30-40 буферов по 70-140 тыс. сэмплов на такт в главном потоке. На слабом телефоне подлагивания. Кэш буферов по ключу `частота+длительность` (всего ~40 уникальных на прогрессию).
5. 🟢 **Мёртвый код отсчёта «1-2-3-4»** — вся логика count-in (countEvents, бейдж, CSS) осталась, но `countEvents` никогда не заполняется. Либо вернуть отсчёт (для джема полезен — успеть взять гитару), либо вычистить.
6. 🟢 **Мелочи:** `rhythmOnlyOn` не сохраняется в localStorage (хотя `metroOn` сохраняется); PWA регистрирует `sw.js`, которого в проекте нет (офлайн не работает, хотя баннер «Установить» обещает приложение).

**Улучшения звука (малой кровью):**
- Даун- и ап-штрихи звучат одинаково по тембру (отличие только в громкости). У живой гитары ап-страйк ярче. Один biquad-фильтр на ап-штрихах.
- Строб между струнами фиксированный (12 мс). Привязать `STRUM` к BPM/жанру — заметный прирост реализма.

## Что это за приложение

**Цель — импровизация и джем на гитаре.** Всё подчинено ей: тренажёр помогает подтягивать теорию и практику именно в импровизации, а не быть справочником. Каждая новая фича проходит тест «помогает ли это прямо во время джема». Ключевая петля обучения: **слышу аккорд → вижу его тоны на грифе → целюсь в них**. Уникальная фишка — подсветка тонов текущего аккорда на грифе во время джем-лупа (учит играть «сквозь аккорды», а не просто по гамме).

**Чего НЕ добавлять (чтобы не перегрузить):** библиотеки табов/ликов, тюнер, запись себя, все семь ладов справочником без применения, теорийные тексты-простыни. Экран и так плотный.

## Аудио-движок

- **`getCtx()`** — ленивый AudioContext, resume при suspended.
- **Master-цепь** — мягкий компрессор-лимитер (ловит пики полифонии) → `masterOutNode` (громкость, слайдер 0–1.5). ⚠️ баг: громкость применяется ПОСЛЕ лимитера (см. баги выше).
- **`makeVoice(midi, sustain, opts)`** (строка 712) — один голос. Два тембра:
  - **acoustic** — Karplus-Strong (`ksBuffer`, строка 691): физически честный щипок, высокие затухают быстрее. ⚠️ генерируется синхронно (см. баги).
  - **distortion** — пила + суб-октава через общий перегруз-бус (`getDistBus`, строка 662): настоящая интермодуляция струн, «жужжащий» сустейн + щелчок медиатора.
- **`chick()`** — глушёный «чик» для фанка/регги. **`metroClick()`** — клик метронома.
- **Штрих боя** (`playStrumStroke`) — down = все ноты, up = верхние снизу-вверх (тише), X = чик. ⚠️ down/up одинаковы по тембру, строб фиксирован 12 мс (см. улучшения звука).

## Джем-луп (Progression Player)

- **`startPlayer(prog, ri, cardEl, rhythmIdx)`** (1430) → `scheduleBar` (1341) планирует такт по аудио-часам (sample-accurate lookahead, без дрожания setTimeout).
- **Подсветка тонов аккорда** (`highlightChordTones`) — при смене такта тоны текущего аккорда загораются среди нот пентатоники/лада на грифе. **Это главная обучающая фича.**
- **Режим «только ритм»** (`rhythmOnlyOn`) — луп на одной тонике, отработка боя. ⚠️ не сохраняется в LS.
- **Метроном** (`metroOn`), **тап-темпо**, **BPM-слайдер** — в раскрывающейся панели ритма у каждой прогрессии. Настройки джема по прогрессии в `jamPrefs` (LS).
- **Мёртвый код:** отсчёт «1-2-3-4» (countEvents) — вся логика есть, но не заполняется (см. баги).

- **`localStorage['penta-trainer-state']`** (`LS_KEY`, строка 844) — `saveState`/`loadState`.

- Гаммы/лады подаются не как справочник, а через настроение + жанр + характерную ноту, с несколькими уровнями глубины (минор+2 ноты → полный лад). Объединили бывшие пункты 3 и «обучение ладам» в один.
- Музыкальные данные проверяются программно, не на глаз (легко ошибиться на полутон).
- Деплой Public-репозиторием (нет ничего чувствительного, проще для Netlify).
